# 🎵 The Music Map: Music Recommendation System

> A content-based music recommendation engine built on the Spotify Global Music Dataset — complete with a live Streamlit web application that displays album artwork for every recommendation.

<br>

## 🖥️ App Preview

Select any song from the dropdown, hit **Play song ▶️**, and the app instantly displays 5 recommended tracks alongside their album posters — fetched live from the Spotify API.

<br>

## 📌 Overview

Navigating a catalogue of 10,000+ songs manually is impractical. This project tackles that problem by building an intelligent recommendation engine that accepts a song title as input and returns five contextually similar tracks — matched by artist identity, album, genre, and release era.

The system is built entirely on **content-based filtering**: no user data, no ratings, no black-box neural networks. Every recommendation is explainable by shared metadata attributes.

<br>

## 🧠 How It Works

```
Raw Dataset  →  Preprocessing  →  Tag Construction  →  Vectorization  →  Cosine Similarity  →  Streamlit App
```

| Stage | What Happens |
|---|---|
| **Preprocessing** | Feature selection, null removal, ASCII filtering, date normalization, whitespace removal |
| **Tag Construction** | Artist + Album + Genre + Date merged into a single text descriptor per track |
| **Stemming** | Porter Stemmer reduces tokens to their morphological roots |
| **Vectorization** | CountVectorizer converts tags into 1,000-dimensional Bag-of-Words vectors |
| **Similarity** | Cosine similarity computed pairwise across all tracks → n×n matrix |
| **Recommendation** | Top-5 most similar tracks retrieved, ranked, and displayed with album art |

## 📊 Dataset

**Source:** Spotify Global Music Dataset  
**Records:** 10,514 tracks | **Columns:** 15 (6 used for modeling)

| Feature Used | Role |
|---|---|
| `track_name` | Query key — input identifier |
| `artist_name` | Primary similarity signal |
| `collection_name` | Album-level grouping |
| `genre` | Dominant content descriptor |
| `release_date` | Era proxy for production style |
| `track_id` | Used to fetch album artwork from Spotify API |

> Columns dropped: `user_rating`, `track_price`, `collection_price`, `content_rating`, `track_time_millis`, `country`, `currency`, `preview_url`, `artwork_url` — none contribute to content similarity.

## 🚀 Usage

### Step 1 — Generate the model files

Open and run `Recommendation_System.ipynb` end-to-end. This produces `musics.pkl` and `similarities.pkl` in the project root.

### Step 2 — Launch the web app

```bash
streamlit run main.py
```

The app opens at `http://localhost:8501` in your browser.

### Step 3 — Get recommendations

1. Select any track from the **"What to play today?"** dropdown
2. Click **Play song ▶️**
3. Five recommended tracks appear with their album artwork, fetched live from Spotify

## 🔬 Algorithm — Cosine Similarity

$$\text{similarity}(A, B) = \cos(\theta) = \frac{A \cdot B}{|A| \times |B|}$$

- Score of **1.0** → identical feature profile  
- Score near **0** → no shared attributes  
- **Scale-invariant** — short and long tags treated fairly

<br>

## ❌ Approaches Considered but Rejected

| Approach | Why Rejected |
|---|---|
| Collaborative Filtering | Requires user interaction logs — unavailable in this dataset |
| Matrix Factorization (SVD) | Requires a user-item rating matrix — not applicable |
| TF-IDF Vectorization | Tags are only 4–8 tokens; IDF weighting gives negligible gain over raw counts |
| Word2Vec Embeddings | Artist/genre names are proper nouns absent from general-purpose embeddings |
| Audio Signal Analysis | No audio files or Spotify API access available |
| Deep Learning (Autoencoders) | Requires GPU + large labelled dataset; poor interpretability for this scope |

<br>

## ⚠️ Known Limitations

- **No personalisation** — recommendations are track-to-track, not user-specific
- **Metadata only** — musically similar tracks with different metadata may score low
- **Static index** — adding new tracks requires rerunning the notebook
- **ASCII-only** — non-English tracks filtered out during preprocessing
- **API dependency** — album artwork display requires a live Spotify API connection
